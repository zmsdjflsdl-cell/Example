# 가상메모리 세그먼테이션 시스템 (Segmentation System)

가상메모리를 관리하는 대표적인 기법 중 하나로, 메모리를 고정된 크기가 아니라 **의미론적 단위(논리적 단위)**인 **세그먼트(Segment)**로 나누어 관리하는 방식입니다. 내부 단편화 문제를 해결하고 코드의 보호 및 공유 측면에서 강점을 가집니다.

---

## 1. 핵심 개념 및 단위
* **세그먼트 (Segment):** 프로세스를 구성하는 논리적 단위 (예: Code, Data, Stack, Main 함수, 서브루틴 등). 각 세그먼트는 **서로 크기가 다릅니다.**
* **세그먼트 테이블 (Segment Table):** 세그먼트의 논리 주소를 물리 주소로 변환하기 위한 테이블. 세그먼트의 크기가 제각각이므로, 시작 주소인 **Base** 외에 크기를 제한하는 **Limit** 정보를 함께 가집니다.

---

## 2. 주소 변환 (Address Translation)
가상 주소(Virtual Address)는 CPU에 의해 **세그먼트 번호($s$)**와 **변위($d$, Offset)**의 쌍으로 표현됩니다.



> **주소 변환 흐름**
> 1. 가상 주소 (VA) = 세그먼트 번호 ($s$) + 변위 ($d$)
> 2. 세그먼트 테이블에서 $s$ 번호에 해당하는 엔트리를 확인하여 제한 크기(**Limit**)와 시작 주소(**Base**)를 가져옴
> 3. **조건 검사:** 변위($d$)가 제한 크기(**Limit**)보다 크면 메모리 침범 인터럽트(Trap) 발생
> 4. 물리 주소 (PA) = 세그먼트 시작 주소 (**Base**) + 변위 ($d$)

---

## 3. 세그먼트 테이블 엔트리(STE) 내부의 주요 필드
페이징과 달리 크기가 가변적이므로 주소 변환 및 제어를 위한 필드가 구체적입니다.
* **기준점 (Base):** 물리 메모리에서 해당 세그먼트가 시작되는 시작 위치(물리 주소).
* **한계치 (Limit):** 해당 세그먼트의 실제 크기 (메모리 보호를 위해 변위 $d$와 비교하는 용도).
* **유효 비트 (Valid Bit):** 해당 세그먼트가 현재 물리 메모리에 로드되어 있는지 여부.
* **보호 비트 (Protection Bit):** 세그먼트 단위로 읽기(R), 쓰기(W), 실행(X) 권한을 다르게 부여 (예: Code 세그먼트는 Read-Only, Stack 세그먼트는 Read/Write).

---

## 4. 세그먼테이션 시스템의 장단점

| 장점 (Pros) | 단점 (Cons) |
| :--- | :--- |
| **내부 단편화 해결:** 프로세스가 필요한 논리적 크기만큼 딱 맞춰 메모리를 할당하므로 내부 단편화가 발생하지 않음. | **외부 단편화 발생:** 세그먼트들의 크기가 서로 다르기 때문에, 메모리 해제와 할당이 반복되면 중간중간 짜투리 공간이 생기는 외부 단편화가 발생함. |
| **보호 및 공유의 용이성:** 의미 단위(함수, 데이터 등)로 나누어져 있기 때문에 권한을 설정하거나(보호), 여러 프로세스가 같은 코드를 공유하기 매우 직관적임. | **메모리 배치 정책 필요:** 가변 크기 메모리 할당이므로 최초 적합(First-Fit), 최적 적합(Best-Fit) 등의 복잡한 배치 전략과 메모리 압축(Compaction) 오버헤드가 발생할 수 있음. |
