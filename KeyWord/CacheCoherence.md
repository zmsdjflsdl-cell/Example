# 캐시 일관성 (Cache Coherence) 핵심 정리

멀티프로세서 시스템에서 여러 개의 프로세서가 독립된 캐시를 가질 때, 동일한 메모리 주소에 대해 서로 다른 데이터를 가지고 있는 '불일치' 문제를 해결하는 원칙입니다.

---

### 1. 주요 발생 원인
* **공유 데이터 수정:** 프로세서 A가 변수 X를 수정했으나, 프로세서 B의 캐시에는 여전히 옛날(Stale) 값이 남아있는 경우.
* **I/O 동작:** DMA(Direct Memory Access) 장치가 메인 메모리 값을 직접 바꿀 때 캐시가 갱신되지 않는 경우.

---

### 2. 일관성 유지를 위한 2가지 메커니즘

#### ① 스누핑 프로토콜 (Snooping Protocol)
* **특징:** 모든 캐시가 공통된 버스(Bus)를 감시(Snooping)하며 메모리 접근을 모니터링합니다.
* **장점:** 구현이 비교적 간단하고 빠릅니다.
* **단점:** 프로세서 수가 많아지면 버스 트래픽 과부하가 발생하여 확장성이 떨어집니다. (소규모 시스템용)

#### ② 디렉토리 프로토콜 (Directory Protocol)
* **특징:** 메모리 블록의 상태 정보를 중앙 디렉토리에서 관리합니다.
* **장점:** 특정 캐시에만 통보하므로 버스 트래픽이 적어 확장성이 좋습니다.
* **단점:** 중앙 관리 장치(Directory) 오버헤드가 발생합니다. (대규모 분산 시스템용)

---

### 3. 대표 프로토콜: MESI 모델
캐시 라인이 가질 수 있는 4가지 상태값입니다.

| 상태 | 명칭 | 설명 |
| :--- | :--- | :--- |
| **M** | Modified (수정) | 데이터가 수정되었으며 메인 메모리와 다름 (이 캐시가 유일한 복사본) |
| **E** | Exclusive (배타) | 메인 메모리와 데이터가 같으며, 오직 이 캐시에만 존재함 |
| **S** | Shared (공유) | 메인 메모리와 데이터가 같으며, 다른 캐시들도 복사본을 가질 수 있음 |
| **I** | Invalid (무효) | 유효하지 않은 데이터 (다른 프로세서에 의해 수정됨) |

---

### 4. 일관성 vs 무결성 (Coherence vs Consistency)
* **Coherence (일관성):** '단일 주소'에 대한 데이터 값이 모든 프로세서에게 어떻게 보일 것인가의 문제.
* **Consistency (무결성/일관성):** '여러 주소'에 대한 읽기/쓰기 순서가 어떻게 보일 것인가의 문제 (Memory Wall 관련).
