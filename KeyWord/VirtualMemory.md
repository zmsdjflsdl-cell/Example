# 가상 메모리 (Virtual Memory) 핵심 요약

가상 메모리는 물리적 메모리(RAM)의 한계를 극복하기 위해 **보조기억장치(HDD/SSD)의 일부를 주기억장치처럼 사용하는 메모리 관리 기법**입니다. 프로세스 전체가 메모리에 올라가지 않아도 실행이 가능하도록 만듭니다.

---

## 1. 핵심 용어 정리

* **가상 주소 (Virtual Address):** 프로세스가 참조하는 논리적 주소 공간. CPU가 생성하는 주소입니다.
* **물리 주소 (Physical Address):** 실제 RAM(주기억장치)의 주소 공간입니다.
* **MMU (Memory Management Unit):** 가상 주소를 물리 주소로 빠르게 변환(Translation)해주는 **하드웨어 장치**입니다.
* **페이지 호환 (Page Fault):** 프로그램이 필요한 페이지가 현재 물리 메모리(RAM)에 없어, 보조기억장치에서 가져와야 하는 상태를 말합니다.



---

## 2. 가상 메모리 관리 기법

### ① 페이징 (Paging)
가상 메모리와 물리 메모리를 **고정된 똑같은 크기**로 나누어 관리하는 방식입니다.

* **단위:** 가상 메모리의 블록은 **페이지(Page)**, 물리 메모리의 블록은 **프레임(Frame)**이라고 부릅니다.
* **주소 변환:** **페이지 테이블(Page Table)**을 두어 페이지가 어떤 프레임에 위치하는지 매핑합니다.
* **특징:** 고정 크기로 나누기 때문에 외부에 남는 공간이 생기는 '외부 단편화'는 없지만, 페이지 내부 공간이 짜투리로 남는 **내부 단편화(Internal Fragmentation)**가 발생할 수 있습니다.

 


### ② 세그먼테이션 (Segmentation)
가상 메모리를 크기가 고정된 것이 아니라, 코드(Code), 데이터(Data), 스택(Stack) 등 **논리적 의미를 가진 가변적인 크기**로 나누어 관리하는 방식입니다.

* **단위:** 크기가 서로 다른 **세그먼트(Segment)** 단위로 분할합니다.
* **주소 변환:** 세그먼트의 시작 주소(Base)와 크기(Limit)를 담은 **세그먼트 테이블**을 사용합니다.
* **특징:** 내부 단편화 문제는 해결되지만, 메모리 중간중간에 애매한 빈 공간이 생겨 활용하지 못하는 **외부 단편화(External Fragmentation)**가 발생할 수 있습니다.



---

## 📌 페이징(Paging) vs 세그먼테이션(Segmentation)

| 비교 항목 | 페이징 기법 (Paging) | 세그먼테이션 기법 (Segmentation) |
| :--- | :--- | :--- |
| **분할 기준** | 일정한 크기 (고정 분할) | 논리적 단위 (가변 분할) |
| **분할 단위** | 페이지 (Page) / 프레임 (Frame) | 세그먼트 (Segment) |
| **단편화 문제** | **내부 단편화** 발생 가능 | **외부 단편화** 발생 가능 |
| **주소 변환** | 페이지 테이블 이용 | 세그먼트 테이블 이용 |
| **장점** | 메모리 할당과 관리가 단순함 | 공유 및 보안(보호) 권한 부여가 용이함 |
