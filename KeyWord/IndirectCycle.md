## 컴퓨터 구조: 간접 사이클(Indirect Cycle) 핵심 정리

### 정의 및 개념
  - 정의: CPU가 명령어를 실행할 때, 오퍼랜드(Operand, 데이터 필드)에 적힌 주소가 실제 데이터의 주소가 아니라
  '실제 데이터가 저장된 곳의 주소를 가리키는 주소(간접 주소)'일 때, 이 실제 유효 주소를 읽어오기 위해 메모리에 한 번 더 접근하는 과정입니다.
  - 위치: 명령어 사이클(Instruction Cycle) 흐름에서 인출 사이클(Fetch Cycle)과 실행 사이클(Execute Cycle) 사이에 위치합니다.

### 명령어 사이클 흐름에서의 위치

  CPU가 명령어를 처리하는 전체 주기 내에서 간접 사이클은 다음과 같이 작동합니다.

 - 인출 사이클 (Fetch): 메모리에서 명령어를 가져옴

 - 간접 사이클 (Indirect): 명령어에 '간접 주소 방식'이 지정되어 있다면, 메모리에 한 번 더 접근해 유효 주소(Effective Address)를 구함

 - 실행 사이클 (Execute): 구한 유효 주소의 데이터를 가지고 명령을 수행함

 - 인터럽트 사이클 (Interrupt): 인터럽트 발생 여부를 확인하고 처리함


### 핵심 키워드 및 메커니즘

| 키워드 | 주요내용 |
| --- | --- |
| 간접 주소 지정 (Indirect Addressing) | 명령어의 주소부가 지정한 메모리 번지에 실제 데이터의 주소가 저장되어 있는 방식. |
| 유효 주소 (Effective Address, EA) | CPU가 실제로 접근하여 데이터를 읽거나 쓰고자 하는 진짜 메모리 주소. |
| 메모리 참조 (Memory Reference) | 주소를 찾기 위해 메모리에 최소 2번 이상 접근해야 하므로 속도가 느려짐. |
| I 비트 (Indirect Bit) | 명령어 형식 내에서 이 명령이 직접(0)인지 간접(1)인지를 구분해 주는 플래그 비트.

### 레지스터 전송문 (Micro-operation)

  > 간접 사이클이 일어날 때 CPU 내부 레지스터(AR: 주소 레지스터, DR: 데이터 레지스터, M: 메모리)에서는 다음과 같은 일이 일어납니다.
  <br><br> $AR \leftarrow IR(Address)$ : 명령어 레지스터(IR)의 주소 부분을 주소 레지스터(AR)로 보냄.
  <br>$AR \leftarrow M[AR]$ : AR이 가리키는 메모리 번지에서 '진짜 주소(유효 주소)'를 읽어와 다시 AR에 저장함.

### 간접 사이클의 장단점

#### 장점(유연성)
 - 기억장치 용량 확대: 명령어 내의 제한된 비트 수(예: 16비트 명령어 중 주소부가 12비트뿐일 때)
 보다 더 큰 메모리 영역(예: 16비트에 해당하는 전체 주소 공간)을 참조할 수 있습니다.
 - 포인터 및 서브루틴 구현 용량: 프로그래밍에서 배열(Array)이나 포인터(Pointer)
 같은 동적 주소 제어를 쉽게 해줍니다.

#### 단점(성능 저하)
 -속도 저하: 데이터를 가져오기 위해 메모리를 한 번 더 읽어야 하므로, 
 직접 주소 지정 방식에 비해 CPU 실행 속도가 떨어집니다.
