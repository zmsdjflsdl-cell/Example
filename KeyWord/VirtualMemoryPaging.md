# 📄 가상메모리 페이징 시스템 (Paging System)

가상메모리를 관리하는 가장 대표적인 기법으로, 가상메모리와 물리메모리를 **고정된 일정한 크기**로 나누어 관리하여 외부 단편화 문제를 해결하는 방식입니다.

---

## 1. 핵심 개념 및 단위
* **페이지 (Page):** **가상메모리**를 고정된 크기로 나눈 블록 (프로세스 관점).
* **프레임 (Frame):** 물리메모리(RAM)를 페이지와 동일한 고정 크기로 나눈 블록.
* **페이지 테이블 (Page Table):** 가상 주소의 페이지가 물리 메모리의 어떤 프레임에 위치해 있는지 매핑해 주는 주소 변환 기법.

---

## 2. 주소 변환 (Address Translation)
가상 주소(Virtual Address)는 CPU에 의해 페이지 번호(p)와  변위(d, Offset)의 쌍으로 표현됩니다.

```text
가상 주소 (VA) = [ 페이지 번호 (p) ] + [ 변위 (d) ]
                      │ 
                (페이지 테이블 매핑)
                      ▼
물리 주소 (PA) = [ 프레임 번호 (f) ] + [ 변위 (d) ]
```

> **주소 변환 흐름**
> 1. 가상 주소 (VA) = 페이지 번호 ($p$) + 변위 ($d$)
> 2. 페이지 테이블 매핑을 통해 페이지 번호($p$)가 물리적인 프레임 번호($f$)로 변환됨
> 3. 물리 주소 (PA) = 프레임 번호 ($f$) + 변위 ($d$)

* **변위($d$):** 페이지 시작점으로부터 얼마나 떨어져 있는지 나타내는 거리로, **주소 변환 전후에도 값이 변하지 않습니다.**

---

## 3. 구조적 특징 및 하드웨어 지원

### 📌 TLB (Translation Lookaside Buffer)
* **개념:** 페이지 테이블 참조 속도를 향상시키기 위해 사용하는 **고속 캐시 메모리(MMU 내부에 위치)**.
* **특징:** 자주 참조되는 페이지 매핑 정보를 저장합니다. 
    * **TLB Hit:** 테이블을 거치지 않고 바로 물리 주소로 변환 (메모리 접근 1번).
    * **TLB Miss:** 메인 메모리의 페이지 테이블을 거쳐야 하므로 주소 변환에 시간이 더 걸림 (메모리 접근 2번).

### 📌 페이지 테이블 엔트리(PTE) 내부의 주요 비트
페이지 테이블의 각 행(Entry)에는 주소 외에도 상태를 나타내는 제어 비트가 포함됩니다.
* **유효 비트 (Valid / Present Bit):** 해당 페이지가 현재 물리 메모리(RAM)에 적재되어 있는지 여부 (`1`이면 RAM에 존재, `0`이면 Swap Area에 존재하여 Page Fault 발생).
* **참조 비트 (Reference / Accessed Bit):** 페이지가 적재된 이후 참조(읽기/쓰기)되었는지 여부 (교체 정책인 NUR에서 활용).
* **변형 비트 (Modified / Dirty Bit):** 페이지가 메모리에 적재된 후 데이터가 수정되었는지 여부 (`1`이면 디스크로 쫓겨날 때 변경 사항을 Write-back 해야 함).
* **보호 비트 (Protection Bit):** 해당 페이지에 대한 읽기(R), 쓰기(W), 실행(X) 권한을 제한.

---

## 4. 페이징 시스템의 장단점

| 장점 (Pros) | 단점 (Cons) |
| :--- | :--- |
| **외부 단편화 해결:** 메모리를 고정 크기로 나누기 때문에 빈 공간이 조각나는 외부 단편화가 발생하지 않음. | **내부 단편화 발생:** 프로세스의 마지막 페이지가 페이지 크기보다 작을 경우 남는 공간이 낭비됨. |
| **공유 및 보호 용이:** 페이지 단위로 권한을 설정하거나 여러 프로세스가 같은 페이지(예: 공유 라이브러리)를 가리키게 할 수 있음. | **메모리 오버헤드:** 페이지 테이블 자체를 저장하기 위한 메모리 공간이 추가로 필요함. |
| **스왑 아웃 용이:** 고정 크기이므로 디스크의 스왑 영역과 데이터를 주고받기 편리함. | **속도 저하 가능성:** TLB 미스 발생 시 주소 변환을 위해 메모리에 두 번 접근해야 하므로 성능이 떨어짐. |
