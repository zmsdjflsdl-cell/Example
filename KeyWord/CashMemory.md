# 캐시 메모리 (Cache Memory) 핵심 키워드 정리

> **핵심 한 줄 요약**
> 캐시 메모리는 **지역성(Locality)**을 이용해 CPU(SRAM)와 메인 메모리(DRAM) 사이의 속도 차이를 극복하는 고속 완충 메모리입니다.

---

## 1. 캐시 메모리의 핵심 개념
* **SRAM (Static RAM)**
  * 캐시 메모리를 구성하는 소자. 메인 메모리(DRAM)보다 회로가 복잡하고 비싸지만 **속도가 압도적으로 빠름**.
* **지역성의 원리 (Principle of Locality)**
  * 캐시 적중률을 높이는 이론적 배경.
  * **시간 지역성 (Temporal Locality):** 최근에 접근한 데이터에 곧 다시 접근할 가능성이 높음 (예: 반복문 변수 `i`).
  * **공간 지역성 (Spatial Locality):** 접근한 데이터의 인근 데이터에 접근할 가능성이 높음 (예: 배열 연속 참조).
* **캐시 적중 및 실패**
  * **Cache Hit:** CPU가 원하는 데이터가 캐시에 있을 때 (속도 빨라짐).
  * **Cache Miss:** 캐시에 데이터가 없어 메인 메모리(RAM)까지 가야 할 때 (속도 느려짐).

---

## 2. 캐시 메모리의 계층 구조 (L1, L2, L3)
* **L1 Cache:** CPU 코어 내부에 위치하며 가장 빠르고 용량이 작음 (Instruction / Data 캐시로 분리).
* **L2 Cache:** CPU 코어별 전용 캐시로, L1의 완충 역할.
* **L3 Cache:** CPU 칩 내부에 위치하며, **모든 코어가 공유**하는 캐시 (용량이 가장 큼).

---

## 3. 캐시 주소 매핑 방식 (Mapping)
* **직접 매핑 (Direct Mapping)**
  * 메모리의 블록을 캐시의 정해진 위치에만 1:1로 고정 매핑. 구조가 단순하고 빠르지만 **캐시 충돌(Conflict Miss)**이 자주 발생.
* **완전 연상 매핑 (Fully Associative Mapping)**
  * 비어있는 캐시 공간 어디든 저장 가능. 충돌은 없지만 데이터를 찾을 때 전체를 다 뒤져야 해서 **검색 회로가 복잡하고 비쌈**.
* **세트 연상 매핑 (Set Associative Mapping)**
  * 위 두 방식을 결합한 대세 방식. 캐시를 여러 세트(Set)로 나누고, 특정 세트 안에서 자유롭게 자리를 잡음 (예: 2-way, 4-way).

---

## 4. 캐시 쓰기 정책 (Write Policy)
* **Write-Through (즉시 쓰기)**
  * 캐시에 데이터를 쓸 때 **메인 메모리에도 동시에** 기록. 데이터 일관성이 보장되지만 쓰기 속도가 느려짐.
* **Write-Back (나중에 쓰기)**
  * 일단 **캐시에만 먼저 쓰고**, 나중에 해당 캐시 블록이 쫓겨날 때(Eviction) 메인 메모리에 업데이트. 속도가 빠르지만 일시적인 불일치 상태가 존재함.

---

## 5. 캐시 교체 알고리즘 (Replacement Algorithm)
* **LRU (Least Recently Used):** 가장 오랫동안 사용되지 않은 데이터를 삭제 (가장 흔히 사용).
* **LFU (Least Frequently Used):** 참조 횟수가 가장 적은 데이터를 삭제.
* **FIFO (First-In, First-Out):** 캐시에 가장 먼저 들어왔던 데이터를 삭제.
