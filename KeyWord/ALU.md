# 키워드 정리
# ALU(산술논리연산장치) 정의
  > CPU 내부에서 덧셈 뺄셈 같은 산술 연산과 
  AND, OR 같은 논리 연산을 수행하는 핵심 디지털 회로
  쉽게 말해 CPU 속에 들어 있는 전용 계산기

  > 산술 연산으로 수치를 계산하고, 논리 연산으로 참과 거짓을 판별하여 
  프로그램의 흐름을 결정함.
  
 - 비트 시프트(Bit Shift)
  > ALU는 단순 계산 외에도 데이터를 좌우로 미는 비트 시프트 연산도 수행한다.
  예를 들어, 이진수 데이터를 왼쪽으로 한 칸 밀면(Shift) 값이 2배가 됨
  이는 곱셈보다 훨씬 빠른 속도로 데이터를 처리하는 비결

# 데이터 처리 흐름 3단계
- 1단계 : 가져오기 (Fetch & Decode): 제어 장치가 “이 두 숫자를 더해라”라는 명령을 내리면, 
ALU는 작업대인 레지스터에서 재료(데이터 A, 데이터 B)를 가져온다.
- 2단계: 실행하기 (Execute): ALU가 가져온 데이터로 덧셈이나 비교 연산을 수행한다.
  >ALU가 계산을 마친 후에는 단순히 결과값만 내놓는 것이 아니다.
  “계산 결과가 0이다”, “음수다”, 혹은 “숫자가 너무 커서 넘쳤다(Overflow)” 
  같은 특이 사항을 플래그 레지스터(Flag Register)에 기록하게 되고,
  이 플래그 정보 덕분에 컴퓨터는 “결과가 0이면 게임을 끝내라” 
  같은 분기(Branch) 처리를 할 수 있음 
  즉, 프로그램의 흐름을 바꾸는 결정적인 단서가 된다.
- 3단계: 저장하기 (Write Back): 계산된 결과값을 다시 레지스터나 메모리에 올려놓습니다.

# CPU와 GPU의 ALU 차이점

- CPU(중앙 처리 장치)
  >ALU 특징: 소수의 매우 똑똑하고 복잡한 ALU 탑재

  > 특기: 순서대로 처리하는 복잡한 논리 연산, 운영체제 실행
- GPU(그래픽 처리 장치)
  > ALU 특징: 수천 개의 단순하고 작은 ALU 탑재

  > 특기: 화면의 수백만 픽셀을 동시에 색칠하는 병렬 처리
