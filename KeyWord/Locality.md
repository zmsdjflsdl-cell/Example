#  컴퓨터 구조: 캐시 메모리 지역성 (Locality) 완벽 정리

## 1. 지역성(Locality)의 본질
* **개념:** CPU가 주기억장치(RAM)의 데이터를 읽어올 때, 특정 주소 영역을 집중적으로 참조하는 경향.
* **등장 배경:** CPU의 연산 속도는 빛처럼 빠른 반면, RAM의 데이터 전송 속도는 느림 (**Memory Wall 현상**). 이 병목 현상을 해결하기 위해 중간에 아주 빠른 소형 메모리인 **캐시 메모리(Cache Memory)**를 배치함.
* **캐시의 작동 원리:** 캐시는 RAM의 전체 데이터를 가질 수 없으므로, **"지역성" 원리에 따라 앞으로 쓰일 것 같은 데이터만 족집게처럼 예측해서 저장**해둠. 예측이 맞으면 **Cache Hit(적중)**, 틀리면 **Cache Miss(실패)**.

## 2. 3대 지역성 핵심 요약

### ① 시간 지역성 (Temporal Locality)
* **한 줄 요약:** "방금 쓴 거 조금 이따가 또 쓴다."
* **설명:** 한 번 참조된 메모리 위치는 아주 가까운 미래에 다시 참조될 가능성이 높음.
* **대표적인 예시:**
  * **반복문 내의 변수:** `for (int i = 0; i < 100; i++)`에서 루프 카운터 변수 `i`.
  * **누적 변수:** 대량의 데이터를 더할 때 사용하는 `sum` 변수.

### ② 공간 지역성 (Spatial Locality)
* **한 줄 요약:** "지금 쓴 거 바로 옆에 있는 것도 곧 쓴다."
* **설명:** 특정 메모리 위치가 참조되면, 그 주소와 인접한 부근의 데이터가 곧바로 참조될 가능성이 높음.
* **대표적인 예시:**
  * **배열(Array):** 메모리에 차례대로 연속해서 저장되므로, `arr[0]`을 읽으면 높은 확률로 `arr[1]`, `arr[2]`도 연속해서 읽음.
  * **캐시 라인(Cache Line):** 이 특성 때문에 캐시는 RAM에서 데이터 1바이트만 딱 가져오지 않고, 보통 64바이트짜리 덩어리(Block)로 묶어서 가져옴.

### ③ 순차 지역성 (Sequential Locality)
* **한 줄 요약:** "특별한 일 없으면 다음 명령어로 넘어간다."
* **설명:** 프로그램 명령어가 분기(Jump/Branch)하지 않는 한, 메모리에 저장된 순서대로 차례차례 실행되는 특성 (넓은 의미에서 공간 지역성에 포함되기도 함).

## 3. 면접/시험 단골 질문: "지역성을 고려한 코드"
개발자가 코드를 어떻게 짜느냐에 따라 캐시 적중률이 극단적으로 달라집니다. 가장 대표적인 예시가 **2차원 배열 탐색**입니다.

### ❌ 캐시 미스가 폭발하는 코드 (열 우선 탐색)
C/C++ 같은 언어는 메모리에 행(Row) 단위로 데이터를 연속 배치합니다. 아래처럼 세로로 건너뛰며 읽으면 공간 지역성이 완전히 깨집니다.
```c
// 메모리를 멀리멀리 건너뛰며 읽기 때문에 Cache Miss 발생!
for (int j = 0; j < 100; j++) {
    for (int i = 0; i < 100; i++) {
        sum += array[i][j]; 
    }
}
