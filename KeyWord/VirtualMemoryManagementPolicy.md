# 가상메모리 관리 정책 (Virtual Memory Management Policy)

가상메모리의 효율적인 관리를 위해 CPU와 OS가 협력하여 메모리에 페이지를 **언제(Fetch), 어디에(Placement), 누구를 내보낼지(Replacement)** 결정하는 정책입니다.

---

## 1. 가져오기 정책 (Fetch Policy)
> **"디스크에서 메모리로 페이지를 언제 가져올 것인가?"**

* **요구 페이징 (Demand Paging)**
    * **개념:** 프로세스 실행 중 실제로 필요한 페이지가 요청(참조)될 때만 메모리에 적재하는 방식.
    * **특징:** 메모리 절약, 초기 적재 시간 감소. 다만, 초기에 *페이지 폴트(Page Fault)*가 자주 발생하여 디스크 I/O 오버헤드가 있을 수 있음.
* **예측 페이징 (Prepaging)**
    * **개념:** 참조될 가능성이 높은 페이지를 주변 탐색 경로를 통해 미리 예측하여 메모리에 적재하는 방식.
    * **특징:** 과도한 페이지 폴트를 방지하고 탐색 시간(Seek Time)을 줄일 수 있으나, 예측이 틀릴 경우 메모리가 낭비됨.

---

## 2. 배치 정책 (Placement Policy)
> **"디스크에서 가져온 페이지/세그먼트를 메모리의 어디에 적재할 것인가?"**
> *(※ 페이징 시스템은 고정 크기이므로 배치 문제가 없으나, 가변 크기인 세그먼테이션 시스템이나 동적 메모리 할당에서 중요하게 다뤄짐)*

* **최초 적합 (First-Fit):** 메모리를 처음부터 탐색하다가 **가장 먼저 발견되는** 빈 공간에 바로 배치. (가장 빠른 속도)
* **최적 적합 (Best-Fit):** 빈 공간 중 프로세스 크기와 차이가 가장 적은, 즉 **단편화(남는 공간)를 최소화하는** 공간에 배치. (탐색 오버헤드가 큼)
* **최악 적합 (Worst-Fit):** 메모리 내에서 **가장 크기가 큰** 빈 공간에 배치. (남는 빈 공간이 커서 추후 다른 프로세스가 재사용하기 유리)

---

## 3. 교체 정책 (Replacement Policy)
> **"메모리가 가득 찼을 때, 어떤 페이지를 디스크(Swap Area)로 내보낼 것인가?"**

| 알고리즘 | 설명 | 특징 및 한계 |
| :--- | :--- | :--- |
| **FIFO** <br>*(First-In, First-Out)* | 메모리에 **가장 먼저 들어온** 페이지를 우선적으로 교체. | 구현이 매우 간단함. 그러나 프레임을 늘려주어도 오히려 페이지 폴트가 증가하는 **Belady's Anomaly** 현상이 발생할 수 있음. |
| **LRU** <br>*(Least Recently Used)* | **가장 오랫동안 사용되지 않은** 페이지를 교체. | 과거의 행적이 미래를 결정한다는 국부성(Locality) 원리에 기반. **가장 대중적이고 효율이 좋아 범용적으로 사용됨.** (카운터나 스택 오버헤드 존재) |
| **LFU** <br>*(Least Frequently Used)* | **참조 횟수가 가장 적은** 페이지를 교체. | 누적 횟수를 기준하기 때문에, 과거엔 많이 썼지만 현재는 쓰지 않는 페이지가 메모리에 계속 머무는 낭비가 생길 수 있음. |
| **NUR / NRU** <br>*(Not Used Recently)* | 최근에 사용되지 않은 페이지를 교체 (LRU 근사 알고리즘). | **참조 비트(R)**와 **변형 비트(M)**를 두어 오버헤드를 줄임. <br>교체 순위: `(R=0, M=0)` $\rightarrow$ `(0, 1)` $\rightarrow$ `(1, 0)` $\rightarrow$ `(1, 1)` |
| **오프티멀 (Optimal)** | 앞으로 **가장 오랫동안 사용되지 않을** 페이지를 교체. | 이론상 최적의 페이지 폴트 횟수를 보장하지만, 미래의 프로세스 참조 흐름을 미리 알 수 없으므로 **구현 불가능 (성능 비교 기준선으로만 사용)**. |

## 💡 핵심 키워드

```text
📌 페이지 폴트 (Page Fault)
   - CPU가 액세스하려는 페이지가 현재 메인 메모리(RAM)에 없어 가상 메모리(디스크)에서 가져와야 하는 상태. 인터럽트가 발생함.

📌 스래싱 (Thrashing)
   - 프로세스가 실행되는 시간보다 페이지 교체(디스크 I/O)를 하는 시간이 더 많아져서 CPU 이용률이 급격히 떨어지는 현상.

📌 워킹 셋 (Working Set)
   - 프로세스가 특정 시간 동안 자주 참조하는 페이지들의 집합을 모델링하여, 이 집합을 통째로 메모리에 상주시켜 스래싱을 방지하는 기법.

📌 국부성 (Locality / 지역성)
   - 프로세스가 실행되는 동안 메모리의 일부 특정 영역을 집중적으로 참조하는 특성.
   - 시간 국부성(최근 참조된 곳을 또 참조)과 공간 국부성(참조된 곳 주변을 참조)으로 나뉨. (LRU의 핵심 근거)
