# 키워드 정리
# Data Bus
 > 데이터 버스(Data Bus)는 컴퓨터 안에서 CPU, 메모리, 
 그래픽카드 등이 실제 데이터(0과 1의 디지털 신호)를 
 서로 주고받는 물리적인 전송 통로(전선 세트)입니다.

 # 핵심 기술 특징

1. 양방향 통행: 데이터를 읽어올 때와 저장할 때 모두 같은 선을 공유해서
  쓰는 왕복 도로입니다.

1. 병렬 전송: 여러 가닥의 전선이 나란히 깔려 있어, 한 번에 대량의 데이터를 동시에 보냅니다. 
  (예: 64비트 컴퓨터 = 64가닥의 선으로 64비트 동시 전송)

1. 대역폭(속도): 이 선의 개수가 많고 신호가 빠를수록 컴퓨터의 전체적인 데이터 처리 속도가 빨라집니다.


# 다른 버스와의 차이
컴퓨터가 일할 때는 항상 아래 3개의 버스가 세트로 움직입니다.

- 주소 버스: 데이터를 어디로 보낼지 '위치(주소)'를 지정 (단방향)
- 제어 버스: 읽기/쓰기 등 어떤 작업을 할지 '명령(신호)'을 전달
- 데이터 버스: 명령과 주소가 정해지면 '실제 데이터(내용물)'를 수송 (양방향)


# 추가 내용

- 데이터 버스의 폭 (Bus Width): 데이터 버스의 선 개수를 뜻합니다. 
이게 바로 컴퓨터가 32비트냐, 64비트냐를 결정하는 기준입니다.
폭이 넓을수록 한 번에 처리하는 데이터양이 많아져 성능이 올라갑니다.

### 병목현상 (Bottleneck)
- CPU가 아무리 빨라도 데이터를 나르는 '데이터 버스'의 속도가 느리면 컴퓨터 전체 속도가 저하되는 현상이 발생합니다. 
(컴퓨터 성능을 볼 때 버스 속도/클록을 같이 보는 이유입니다.)
