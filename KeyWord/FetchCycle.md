#  명령어 사이클: 인출 사이클 (Fetch Cycle) 핵심 요약

인출 사이클은 **메모리(주기억장치)에 있는 명령어를 CPU로 가져와서(Fetch) 다음 명령어를 준비하는 단계**.

## 1. 인출 사이클 3단계 마이크로 연산


| 순서 | 마이크로 연산 (Micro-operation) | 핵심 설명 (무슨 일이 일어나는가?) |
| :---: | :--- | :--- |
| **1단계** | `MAR ← PC` | 프로그램 카운터(PC)에 저장된 **다음에 실행할 명령어의 주소**를 메모리 주소 레지스터(MAR)로 보낸다. |
| **2단계** | `MBR ← M[MAR]` <br> `PC ← PC + 1` | **[동시 진행]** <br>1. MAR가 가리키는 메모리 주소(`M[MAR]`)에서 명령어를 읽어 메모리 버퍼 레지스터(MBR)에 저장한다. <br>2. CPU는 다음 명령어를 가리키기 위해 프로그램 카운터(PC)의 값을 1 증가시킨다. |
| **3단계** | `IR ← MBR` | MBR에 임시 저장되어 있던 명령어를 **현재 실행할 명령어 레지스터(IR)**로 이동시켜 인출을 완료한다. |

<br>

## 2. 인출 사이클 핵심 키워드 체크리스트

* **주소의 출발점은 언제나 PC:** 인출 사이클이 시작될 때 명령어의 위치를 알고 있는 곳은 오직 **PC(프로그램 카운터)**뿐이야.
* **PC의 자동 증가 (PC + 1):** 명령어를 메모리에서 '읽어오는 동작'과 동시에 PC 값이 늘어나. 그래야 다음 루프 때 다음 명령어를 가져올 수 있으니까! (단, 명령어의 크기가 1워드일 때 기준)
* **메모리 접근의 관문은 MAR와 MBR:** 메모리에서 데이터를 가져올 때는 무조건 주소를 버스에 싣는 **MAR**와 데이터를 받아내는 **MBR**을 거쳐야 해.
* **최종 목적지는 IR:** 인출된 명령어가 최종적으로 IR(명령어 레지스터)에 들어가야 인출 사이클이 끝나고, 다음 단계인 해독/실행으로 넘어갈 수 있어.
