# 컴퓨터 구조: 명령어 사이클 (Instruction Cycle) 핵심 요약

## 1. 명령어 사이클 4단계 단계별 정리

| 단계 (Cycle) | 개념 요약 | 핵심 마이크로 연산 (Micro-operation) |
| :--- | :--- | :--- |
| **인출 (Fetch)** | 메모리에서 명령어를 읽어 CPU(IR)로 가져오는 단계 | 1) MAR ← PC <br> 2) MBR ← M[MAR], PC ← PC + 1 <br> 3) IR ← MBR |
| **간접 (Indirect)** | 간접 주소 방식일 때, 실제 데이터가 있는 **유효 주소**를 찾으러 메모리에 재접근하는 단계 | 1) MAR ← IR(Addr) <br> 2) MBR ← M[MAR] <br> 3) IR(Addr) ← MBR |
| **실행 (Execute)** | 명령어의 종류(ADD, LOAD 등)를 판별하고 실제로 연산을 수행하는 단계 | 명령어 종류에 따라 다름 (ALU 및 제어장치 구동) |
| **인터럽트 (Interrupt)** | 명령어 실행 중 예기치 못한 상황 발생 시, 현재 PC를 저장하고 복귀 주소로 제어를 옮기는 단계 | 복귀 주소(PC)를 스택/메모리에 저장 후 인터럽트 서비스 루틴(ISR) 시작 주소 복사 |

<br>

## 2. 필수 암기 핵심 레지스터

| 레지스터 (Register) | 약자 | 핵심 역할 (한 줄 요약) |
| :--- | :--- | :--- |
| **프로그램 카운터** | **PC** | **다음에 실행할** 명령어의 메모리 주소를 보관 |
| **메모리 주소 레지스터** | **MAR** | 데이터를 읽거나 쓸 **메모리의 주소**를 임시 보관 |
| **메모리 버퍼 레지스터** | **MBR** | 메모리에서 읽어온 데이터나 **쓸 데이터**를 임시 보관 |
| **명령어 레지스터** | **IR** | **현재 실행 중인** 명령어를 보관 |
| **누산기** | **AC** | 연산의 **중간 결과**를 임시로 저장하는 레지스터 |

<br>

## 3. 헷갈리기 쉬운 핵심 키워드 매칭 매커니즘

* **PC -> MAR:** 항상 시작 주소는 PC에서 MAR로 이동 (MAR ← PC)
* **M[MAR] -> MBR:** 메모리에서 읽어온 데이터는 무조건 버퍼 레지스터(MBR)를 거침
* **MBR -> IR:** 최종 인출 완료된 명령어는 가차없이 명령어 레지스터(IR)에 안착
* **간접 사이클의 목적:** 오직 하나, 진짜 데이터 주소인 **"유효 주소(Effective Address) 찾기"**

## 4. 약자 설명
| 용어 | 풀네임 (Full Name) | 쉬운 설명 (비유) | 역할 요약 |
| :--- | :--- | :--- | :--- |
| **MAR** | **M**emory **A**ddress **R**egister <br>(메모리 주소 레지스터) | **"택배 보낼 주소 적는 라벨"** | CPU가 메모리에서 데이터를 읽어오거나 보낼 때, **"몇 번지 메모리를 쓸지" 그 주소를 잠시 기억**하는 저장소. |
| **MBR** | **M**emory **B**uffer **R**egister <br>(메모리 버퍼 레지스터) | **"택배 상자를 잠시 두는 현관문 앞"** | 메모리와 CPU 사이에 주고받는 **실제 데이터나 명령어를 잠시 담아두는 버퍼(완충) 공간**. |
| **IR** | **I**nstruction **R**egister <br>(명령어 레지스터) | **"지금 읽고 있는 요리책 페이지"** | 메모리에서 방금 막 인출(Fetch)해온, **"지금 당장 실행할 명령어"를 보관**하는 곳. |
| **Addr** | **Addr**ess <br>(주소) | **"번지수 / 위치"** | 명령어 안에서 **데이터가 저장된 메모리의 위치(방 번호)**를 가리키는 말이야. 보통 `IR(Addr)`라고 쓰면 "현재 명령어 레지스터에 있는 명령어 중 '주소'를 나타내는 부분"이라는 뜻. |
