## 1. 제어 버스(Control Bus) 정의
CPU와 주변 장치 간에 제어 신호(Control Signals)를 전송하기 위한 물리적인 선들의 집합입니다. 
시스템 버스(System Bus)를 구성하는 3대 요소 중 하나로, 하드웨어 장치들의 동작을 동기화하고 관리합니다.

## 2. 주요 제어 신호

| 신호 명칭 | 설명 |
| --- | --- |
| Memory Read/Write | 메모리에 데이터를 쓰거나 읽으라는 명령 |
| I/O Read/Write | 입출력 장치(키보드, 모니터 등)에 데이터를 입출력하라는 명령 |
| Interrupt (인터럽트) | 하드웨어 장치가 CPU에 긴급 상황이나 작업 완료를 알림 |
| Bus Request/Grant | 특정 장치가 버스 사용권을 요청하고, CPU가 이를 허가함 |
| Clock (클록) | 시스템 전체의 타이밍을 맞추기 위한 주기적인 신호 |
| Reset | 시스템의 모든 상태를 초기화함 |

## 3. 제어 버스의 중요성

  - 데이터 무결성 유지: 주소 버스와 데이터 버스가 올바른 시점에 작동하도록 제어하여 데이터가 엉뚱한 곳으로 가거나 유실되는 것을 방지합니다.
  - 시스템 효율성: 인터럽트 신호를 통해 CPU가 장치들을 일일이 감시(Polling)하지 않아도 되게 하여 전체 처리 속도를 높입니다.
  - 충돌 방지: 여러 장치가 동시에 데이터를 보내려 할 때 순서를 정해주는 '버스 중재' 역할을 수행합니다.

## 4. 다른버스와의 차이점

| 구분 | 주소 버스 (Address Bus) | 데이터 버스 (Data Bus) | 제어 버스 (Control Bus) |
| --- | --- | --- |
| 역할 | 데이터의 위치(주소) 지정 | 실제 데이터 이동 | 제어 및 상태 신호 전달 |
| 방향성 | 단방향(CPU -> 장치) | 양방향 | 단방향 및 양방향 혼재 |
| 비유 | 주소지 | 화물 | 신호등 및 관제탑 |

핵심 요약: 제어 버스는 컴퓨터 내부의 모든 장치가 제시간에 올바른 작업을 수행하도록 관리하는 시스템의 중추적인 통제 통로입니다
