#  캐시 메모리 매핑 (Cache Memory Mapping) 핵심 요약

캐시 메모리는 주 기억장치(Main Memory)에 비해 크기가 매우 작기 때문에, 메인 메모리의 데이터를 캐시의 어느 위치에 저장할지 결정하는 **매핑(Mapping)** 과정이 필수적입니다.

---

## 1. 직접 매핑 (Direct Mapping)
메인 메모리의 블록을 캐시의 **정해진 하나의 라인**에만 매핑하는 가장 단순한 방식입니다.

* **매핑 공식:** `캐시 라인 번호 = (메인 메모리 블록 번호) mod (캐시 라인 총 개수)`
* **주소 구조:** `[ 태그(Tag) | 라인 인덱스(Index) | 블록 오프셋(Offset) ]`
* **장점:** 구현이 매우 단순하고, 위치가 지정되어 있어 **검색 속도가 빠름**.
* **단점:** 서로 다른 메모리 블록이 동일한 캐시 라인을 공유하므로, 번갈아 참조될 때 캐시 교체가 빈번하게 발생하는 **극심한 캐시 미스(Conflict Miss)**가 발생함.



---

## 2. 연관 매핑 (Fully Associative Mapping)
메인 메모리의 블록이 캐시의 **어느 라인이든 빈 곳이면** 어디나 들어갈 수 있는 방식입니다.

* **주소 구조:** `[ 태그(Tag) | 블록 오프셋(Offset) ]` (인덱스 개념이 없음)
* **장점:** 빈 공간을 자유롭게 활용하므로 충돌 미스가 적고, **캐시 적중률(Hit Ratio)이 높음**.
* **단점:** 캐시에서 데이터를 찾을 때 모든 라인의 태그를 병렬로 검사해야 하므로, **하드웨어 회로(CAM)가 복잡하고 가격이 비싸며 탐색 시간이 길어질 수 있음**.



---

## 3. 세트 연관 매핑 (Set Associative Mapping)
직접 매핑과 연관 매핑의 장점을 결합한 절충안으로, **현대 CPU에서 가장 많이 사용**되는 방식입니다. 캐시를 여러 개의 세트(Set)로 나누고, 각 세트 내부에 여러 개의 라인을 둡니다.

* **구조:** 1개 세트에 $N$개의 라인이 들어가면 **N-Way 세트 연관 매핑**이라고 부름.
* **매핑 방식:** 세트 위치는 직접 매핑(Index)으로 찾고, 세트 내부에서는 연관 매핑(비어있는 아무 라인이나)으로 저장.
* **주소 구조:** `[ 태그(Tag) | 세트 인덱스(Set Index) | 블록 오프셋(Offset) ]`
* **특징:** 직접 매핑의 충돌 문제와 연관 매핑의 비용 문제를 모두 적절히 해결함.



---

## 📌 한눈에 보는 핵심 키워드 비교

| 비교 항목 | 직접 매핑 (Direct) | 연관 매핑 (Fully) | 세트 연관 매핑 (Set) |
| :--- | :--- | :--- | :--- |
| **저장 위치** | 지정된 단 하나의 라인 | 캐시 내 모든 라인 가능 | 지정된 세트 내 모든 라인 가능 |
| **하드웨어 복잡도**| 낮음 (단순함) | 매우 높음 | 중간 (적절함) |
| **적중률 (Hit)** | 낮음 (충돌 잦음) | 높음 | 높음 |
| **검색 속도** | 매우 빠름 | 느림 (전체 탐색 필요) | 중간 |
| **주요 용도** | 단순한 임베디드 시스템 | 소규모 특수 목적 캐시 | **현대 범용 CPU (L1, L2, L3)** |
