## 1. 실행 사이클의 정의
**실행 사이클(Execution Cycle)**이란, 
CPU의 명령어 사이클(Instruction Cycle) 중 두 번째 단계로, 
인출(Fetch)된 명령어를 해독(Decode)하고 연산 장치(ALU)를 통해 데이터 처리, 메모리 저장 등의 실제 작업을 수행하는 과정이다.

## 핵심 동작 프로세스 (4가지 기본 흐름)
명령어 종류에 따라 세부 과정은 다르지만, 일반적으로 다음 4단계의 흐름을 가집니다.

1. 명령어 해독 (Decode): 명령어 레지스터(IR)에 있는 명령어의 연산 코드(Op-code)를 제어장치가 해석합니다.

2. 오퍼랜드 인출 (Operand Fetch): 연산에 필요한 데이터(오퍼랜드)가 메모리나 레지스터에 있다면, 그 주소를 찾아 데이터를 가져옵니다.

3. 연산 수행 (Execute): 산술논리연산장치(ALU)를 통해 실제 산술(+, - 등) 또는 논리(AND, OR 등) 연산을 수행합니다.

4. 결과 저장 (Store): 연산된 결과를 목적지 레지스터나 주기억장치(RAM)에 저장합니다.

## 핵심 키워드 및 관련 레지스터
실행 사이클을 설명할 때 반드시 들어가야 하는 필수 키워드와 레지스터입니다.

- IR (Instruction Register, 명령어 레지스터): 페치 사이클에서 가져온 현재 실행할 명령어를 보관하는 곳입니다. 실행 사이클은 IR의 내용을 읽으면서 시작됩니다.

- 제어장치 (Control Unit): IR의 명령어를 해독하여 CPU 내부의 각 장치에 필요한 제어 신호(Control Signal)를 보냅니다.

- AC (Accumulator, 누산기): 연산할 때 데이터나 연산 후의 중간 결과 값을 임시로 저장하는 핵심 레지스터입니다.

- MAR / MBR: 메모리에 접근하여 데이터를 읽거나(오퍼랜드 인출) 쓸 때(결과 저장) 각각 주소와 데이터를 거쳐 가게 하는 징검다리 역할을 합니다.

## 주요 명령어별 실행 사이클 예시
어떤 명령어가 들어오느냐에 따라 실행 사이클의 레지스터 이동 경로(Micro-operation)가 달라집니다. 가장 대표적인 3가지를 정리해 드릴게요.

#### LOAD 명령어 (메모리의 데이터를 AC로 가져오기)
MAR <- IR(Addr) : 명령어의 주소 부분을 MAR로 보냄
<br>MBR <- M[MAR] : 해당 메모리 주소의 데이터를 MBR로 읽어옴
<br>AC <- MBR : MBR의 데이터를 누산기(AC)에 저장

#### ADD 명령어 (메모리의 데이터를 기존 AC 값과 더하기)
MAR <- IR(Addr) : 더할 데이터가 있는 메모리 주소를 MAR로 보냄
<br>MBR <- M[MAR] : 해당 주소의 데이터를 MBR로 읽어옴
<br>AC <- AC + MBR : 기존 AC 값과 MBR 값을 ALU로 더해 다시 AC에 저장

#### STORE 명령어 (AC의 결과를 메모리에 저장하기)
MAR <- IR(Addr) : 저장할 메모리 주소를 MAR로 보냄
<br>MBR <- AC : AC에 있는 결과 값을 MBR로 옮김
<br>M[MAR] <- MBR : MBR의 값을 지정된 메모리 주소에 기록
